- [ ] Runtime test harness `ensure_binary_built()` in `runtime/tests/exec_subprocess.rs` returns early when `target/release/gvrn` merely exists, without checking its version — so after a version bump a stale release binary from the prior version makes the version-assertion test (`exec_emits_terminal_error_envelope_on_unparseable_command_file`, which asserts the error envelope's `runtime-version` equals `env!("CARGO_PKG_VERSION")`) fail locally against the old version string. CI is unaffected (a fresh checkout has no prior `target/release` binary, so it builds fresh). Harden: run `cargo build --release` unconditionally (incremental — a fast no-op when the binary is already current) or gate the early-return on a version match, so a stale binary can never be tested.
