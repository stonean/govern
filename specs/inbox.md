- Architectural exploration: re-frame the runtime's LLM extension points (`writeCode`, `writeSpecBody`, `assessSpecQuality`, future multi-turn points) as named Anthropic-style Skills the host loads at the seam, rather than ad-hoc JSON envelopes. Potential benefits: structural cache anchoring (Skills are a natural cache boundary); third-party hosts integrate against an emerging Skills protocol instead of govern-specific JSON; `constitution-excerpts` becomes a bundled resource rather than an inline string array. Speculative — depends on Anthropic's Skills protocol stabilizing and is a larger redesign than 022's current scope. Revisit after the writeCode payload-bundling scenario on 022 ships and the cache-anchored shape proves out the pattern. Surfaced 2026-05-19 during runtime-improvement investigation.
- Latent bug: `compute-review-scope`'s `read_plan_affected` (`runtime/src/primitives/compute_review_scope.rs:133`) parses the plan's `## Affected Files` section as a bullet list (dash + backticked path), but every real plan and the `/gov:plan` writeSpecBody template emit a Markdown table (pipe-delimited File / Action / Purpose columns), which `payload.rs::parse_affected_files` (writeCode) parses correctly. The two Affected-Files parsers disagree on format, so for real (table-format) plans `read_plan_affected` returns empty and the review scope silently drops the plan-affected half — scope collapses to modified-since alone, defeating the "plan Affected Files unioned with files modified since diff-base, larger wins" contract from the review-runtime-acceleration scenario. Fix: unify on the table parser (reuse `payload::parse_affected_files`) or accept both forms, and add a table-format case to `compute_review_scope::tests`. Surfaced 2026-07-06 during task 46b (the review-basic exec fixture's table-format plan yielded an empty scope).
