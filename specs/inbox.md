- **REUSE-002: refactor `find_in_progress_commit` to use `read_blob_from_tree`:** `runtime/src/primitives/check_stuck.rs:137-141` keeps an inline blob-lookup pattern (`tree.get_path(...)?.find_blob(...)?.content().str::from_utf8(...)`) duplicating what the newly-added `read_blob_from_tree` helper does. The bug-fix commit deliberately did not couple the refactor with the behavior change. Fix: replace the inline match with `read_blob_from_tree(repo, &tree, spec_rel)?.and_then(|c| extract_status(&c).map(str::to_string))`. Existing tests cover the function, so behavior change is verifiable. (Origin: spec 022 review of check-stuck-tasks-md-advancement, 2026-05-18.)

- **`/audit` Family 4 — re-template `/gov:` literals across framework command sources:** `scripts/audit/placeholder-roundtrip.sh` correctly flags every framework/commands/*.md file containing literal `/gov:` (and bare `gov:`) where `/{project}:` placeholders belong. The framework's command sources are pre-substituted for the gov project rather than properly templated; adopters scaffolding into e.g. an "anvil" project receive files that mention `/gov:foo` instead of `/anvil:foo`. Fix is a mechanical sweep across `framework/commands/*.md`, `.claude/commands/gov/*.md` (via gen-claude-commands.sh regen), `framework/bootstrap/govern.md`, and probably `framework/bootstrap/configure/*.md`. Ships as its own feature spec since it touches every command file. Alternative: declare `/gov:` canonical and tighten Family 4 to skip those backticks — also viable, smaller surface, but contradicts spec 012's multi-agent contract. Decide which path before scoping the fix. (Origin: spec 026 /audit v1 advisory findings, 2026-05-18.)

- **`/audit` Family 8 — past-tense rewrites for old-name references in done specs:** `scripts/audit/introducing-drift.sh` flags ~9 done specs (011, 014, 017, 020, 021, 022, 023, 024, 000) that retain backticked references to renamed commands (`/capture`, `/elaborate`, `/validate`, `gov-rt:`) in current-tense prose. Each rewrite is a small `/gov:ask` cycle on the affected spec — append a clarify-question, accept the past-tense rewrite. Spec body language shifts from "A new `/capture` command provides..." to "`/capture` provided..." (or replaces with the new name's past-tense form where applicable). Per the design, this is intentionally maintainer-paced — the audit surfaces it, the maintainer addresses it organically when they touch the affected specs for other reasons. (Origin: spec 026 /audit v1 advisory findings, 2026-05-18.)

- **REUSE-001: extract `scripts/audit/lib.sh` for shared boilerplate:** Each of the nine `scripts/audit/*.sh` family check scripts repeats ~10 lines of identical setup (`set -uo pipefail`, `ROOT=...`, `cd "$ROOT"`, `drift=0`, the `emit()` function with pipe-separated output). A shared `scripts/audit/lib.sh` exposing `audit_emit FAMILY LOCATION MESSAGE FIX` plus the ROOT/cd setup would eliminate ~90 lines of duplication. Trade-off: shared lib couples the scripts and reduces stand-alone-invocation ergonomics (`bash scripts/audit/X.sh` would require lib.sh on a relative path). Documented in `scripts/audit/README.md` as the v1 trade-off; extracting is optional cleanup, not a blocker. (Origin: spec 026 review SHOULD finding, 2026-05-18.)

- **QUALITY-001: refactor `flush_step` in primitive-promotion-candidates.sh to avoid caller-scope mutation:** `scripts/audit/primitive-promotion-candidates.sh`'s `flush_step()` function reads and mutates seven caller-scope variables (`step_start_line`, `step_buffer`, `step_has_primitive`, `step_has_llm_marker`, `step_has_ignore`, plus emit's `drift`). Bash semantics make this work (functions inherit caller scope by default), but the pattern is fragile to refactor — adding a `local` keyword anywhere in the function would silently break it. Confidence 70% — works as tested but brittle. Fix: convert the per-file loop into a function that returns the step list, then iterate the returned list to apply flush logic. (Origin: spec 026 review low-confidence finding, 2026-05-18.)
