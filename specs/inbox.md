# Inbox

- Runtime `fetch-archive` DNS-rebinding TOCTOU: `validate_fetch_url` resolves the host and rejects internal addresses, then `reqwest` re-resolves independently at connect time, so a host that resolves to a public IP during validation and an internal IP at connect could slip past the SSRF screen. Inherent to BE-INPUT-007's prescribed resolve-then-block design (its Verification explicitly accepts resolution-time blocks), so not a review violation — but a hardened form (custom resolver / pinned socket addr passed to the connector, or connect-time re-check) would close the window. Low priority. Surfaced 2026-07-11 during the 0.18.0 fix-verification pass.

- Architectural exploration: re-frame the runtime's LLM extension points (`writeCode`, `writeSpecBody`, `assessSpecQuality`, future multi-turn points) as named Anthropic-style Skills the host loads at the seam, rather than ad-hoc JSON envelopes. Potential benefits: structural cache anchoring (Skills are a natural cache boundary); third-party hosts integrate against an emerging Skills protocol instead of govern-specific JSON; `constitution-excerpts` becomes a bundled resource rather than an inline string array. Speculative — depends on Anthropic's Skills protocol stabilizing and is a larger redesign than 022's current scope. Revisit after the writeCode payload-bundling scenario on 022 ships and the cache-anchored shape proves out the pattern. Surfaced 2026-05-19 during runtime-improvement investigation. **On hold per user 2026-07-11.**
- Runtime dependency currency: bump rmcp from the 1.x pin (Cargo.toml `rmcp = "1"`, resolves 1.8.0) to 2.x (2.2.0 available). Breaking major — the MCP server's `#[tool]` methods, ToolRouter, ServiceExt, and transport::stdio all sit on rmcp's macro/router API that reworked between 1.x and 2.x, so it needs a port + full test pass (incl. tests/mcp.rs) + its own gvrn-v tag (the next minor after the current release). Non-urgent: 1.8.0 is functional, no advisory; cargo just notes the newer major on install. Read rmcp's 2.x migration guide to scope before starting. **Carries spec 022 task 65** ([mcp-arg-unknown-field-strictness](022-deterministic-runtime/scenarios/mcp-arg-unknown-field-strictness.md)): MCP unknown-field rejection needs a custom `Parameters`-style wrapper at exactly the rmcp param layer this migration reworks, so build the wrapper against 2.x during the port and close task 65 in the same change — check its box in `022-deterministic-runtime/tasks.md` and move the scenario's open question to Resolved. Surfaced 2026-07-11 post-0.18.0 release.
- Runtime `append-inbox` appends comment-blind while it counts and dedups comment-aware (`append_bullet` in `runtime/src/primitives/append_inbox.rs` vs `iter_bullets` / `count_inbox_bullets` in `primitives/mod.rs`). If `inbox.md` ends inside an *unclosed* `<!--` HTML comment (no closing `-->`, i.e. malformed), the new `- [ ] {text}` lands inside the comment region, and `count_inbox_bullets` then skips it — the item is both invisible and undercounted. Requires malformed input (the shipped template closes its comment before the trailing bullet), so low priority; the asymmetry — comment-aware read, comment-blind write — is the latent smell. A hardened `append_bullet` would find the last position outside any comment/fence to append after. Surfaced 2026-07-12 during the command-runtime alignment review (gvrn 0.20.0).
- Runtime `write-review` renders the *known* waiver fields (`rule`, `file`, `reason`, `waived-at`, `waived-by`) through `yaml_scalar`, which does not quote a value that would re-parse as a non-string (a bare `1234` / `true` / `null`). The open-schema *extra* fields were hardened to quote these via `yaml_string` (gvrn 0.20.0), but the known-field path retains the gap: a waiver whose `reason` (or other known field) were a bare-numeric or bool-like string would render unquoted, then fail the next `RawWaiver` parse (a loud error, not silent corruption). Practically unreachable — reasons are justification text, `waived-by` is an email, `rule` / `file` are IDs and paths — but the inconsistency sits right next to the extras path that was just fixed. Routing known-field values through `yaml_string` too would close it; verify no golden churn on timestamp-shaped values (`waived-at`) first. Surfaced 2026-07-12 during the command-runtime alignment review (gvrn 0.20.0).
