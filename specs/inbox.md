# Inbox

- Architectural exploration: re-frame the runtime's LLM extension points (`writeCode`, `writeSpecBody`, `assessSpecQuality`, future multi-turn points) as named Anthropic-style Skills the host loads at the seam, rather than ad-hoc JSON envelopes. Potential benefits: structural cache anchoring (Skills are a natural cache boundary); third-party hosts integrate against an emerging Skills protocol instead of govern-specific JSON; `constitution-excerpts` becomes a bundled resource rather than an inline string array. Speculative — depends on Anthropic's Skills protocol stabilizing and is a larger redesign than 022's current scope. Revisit after the writeCode payload-bundling scenario on 022 ships and the cache-anchored shape proves out the pattern. Surfaced 2026-05-19 during runtime-improvement investigation.
- Runtime path-validation consistency: `validate_no_traversal` is applied by some path-taking primitives but not the destructive one (`enforce-manifest` accepts an arbitrary absolute `directory` + `glob` + `recursive` with an empty `expected` list — a filesystem-wide delete loop), not `apply-manifest`'s manifest-entry `source`/`dest` (fetched-artifact class, traversal-writes outside the target root), and not `merge-managed-block`/`merge-permissions` path args. Apply the helper uniformly. Surfaced 2026-07-11 during runtime review.
- Runtime `check-stuck` assumes linear first-parent history: the in-progress transition commit on a merged side branch is never reached (inflated `commit_count`, possible false `stuck: true`), `find_in_progress_commit` tracks `previous_status` across a topological walk (phantom transitions on branchy history), and its hand-rolled `extract_status` misses CRLF close fences (duplicate of `split_frontmatter`, less correct). Surfaced 2026-07-11 during runtime review.
- Runtime `fetch-archive`: calls reqwest's blocking client from the async MCP handler (documented panic in debug builds, blocked tokio worker in release — wrap in `spawn_blocking` or use the async client); silently truncates bodies over the 256 MiB cap and writes the truncated archive with a matching computed sha; module doc claims the archive is written on checksum mismatch but the code errors before writing. Surfaced 2026-07-11 during runtime review.
- Runtime `extract-archive`: zip symlink entries are materialized as regular files whose content is the link-target path (tar path correctly skips symlinks; module doc claims both do); the 0o7777 mode mask preserves setuid/setgid/sticky bits from untrusted archives — mask to 0o777. Surfaced 2026-07-11 during runtime review.
- Runtime `write-review` writes `review.md` before reading/parsing spec frontmatter, so a malformed spec halts after the first write — fresh review.md, stale spec `review:` block. Compute both writes first, then write. Surfaced 2026-07-11 during runtime review.
- Runtime `substitute-templates` lands outputs at the 0600 tempfile mode, discarding source modes (the AGENTS.md `write_atomic_bytes` gotcha) — an executable in the source tree emits non-executable. Mirror source permissions like `apply-manifest`/`extract-archive`. Surfaced 2026-07-11 during runtime review.
- Runtime input sanitization: `create-scenario` does not YAML-escape `"`/`\` in the LLM-supplied `section` (produces unparseable frontmatter); `append-task` interpolates `title`/`done-when`/`body` into tasks.md without newline sanitization (structure injection). Surfaced 2026-07-11 during runtime review.
- Runtime parser step numbering: ordered-list `start` values are discarded and `<!-- audit:ignore-promotion -->` markers split markdown lists, so all six rewritten commands parse with wrong step labels (status.md's six steps all parse as step 1) — `progress.step` and gate names sent to hosts are wrong/duplicated, and data-model's `StepNumber` contract is not honored. Surfaced 2026-07-11 during runtime review.

Document the convention in spec 022 and/or make a `gate-confirm` primitive step inherently blocking. Surfaced 2026-07-11 during runtime review.
