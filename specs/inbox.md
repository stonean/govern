# Inbox

- Architectural exploration: re-frame the runtime's LLM extension points (`writeCode`, `writeSpecBody`, `assessSpecQuality`, future multi-turn points) as named Anthropic-style Skills the host loads at the seam, rather than ad-hoc JSON envelopes. Potential benefits: structural cache anchoring (Skills are a natural cache boundary); third-party hosts integrate against an emerging Skills protocol instead of govern-specific JSON; `constitution-excerpts` becomes a bundled resource rather than an inline string array. Speculative — depends on Anthropic's Skills protocol stabilizing and is a larger redesign than 022's current scope. Revisit after the writeCode payload-bundling scenario on 022 ships and the cache-anchored shape proves out the pattern. Surfaced 2026-05-19 during runtime-improvement investigation.
- Runtime error-message strings hardcode `specs/` even when `[paths] specs-root` is renamed (spec 040). The `#[error(...)]` templates in `runtime/src/primitives/mod.rs` (e.g. `FeatureNotFound` → "feature directory not found: specs/{feature}", plus `NoSpecHistory`, `TaskNotFound`, `CriterionOutOfRange`, `StatusMismatch`, `StatusFieldMissing`, `MissingSpecFile`) emit `specs/{feature}` regardless of the configured root, so a renamed-root adopter sees a misleading path on error. Cosmetic only — path *resolution* is correct (spec 040 task 2); the variants would need to carry the resolved root/path to format it accurately. Surfaced 2026-06-29 implementing spec 040.
- Cross-service reference resolution assumes the *referenced* service uses `specs/` (spec 040 / spec 030). `gen-cross-service-refs.sh`'s URL matcher (`/specs/NNN-slug/`) and the runtime `resolve-references` checkout read both target another repo's layout; spec 040 made THIS repo's enumeration root-aware but a referenced service that renamed ITS spec root would not resolve. Edge case (both repos must rename and reference each other); deferred from spec 040's scope. The runtime `resolve-references` checkout side already reads the checkout's own `.govern.toml` (correct), but the harvest-time URL matcher in the generator is still `specs/`-only. Surfaced 2026-06-29 implementing spec 040.
